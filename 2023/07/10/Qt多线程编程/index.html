Qt多线程编程

前言

  • 默认的线程在Qt中成为窗口线程，也叫主线程，负责处理窗口事件或者窗口控件数据的更新
  • 子线程负责后台的业务处理，子线程中不能对窗口对象做任何操作，这些事情需要交给窗口线程处理
  • 主线程和子线程之间如果要进行数据的传递，需要使用Qt中的信号机制

1.线程类QThread

QThread 类提供了以下一些常用的接口和方法：

  1. QThread()：构造函数，用于创建一个新的 QThread 对象。
  2. void start()：启动线程。该方法会自动调用 run() 方法，在新的线程中执行指定的代码。
  3. void run()：线程的入口函数，需要在子类中重写。在该函数中，你可以编写需要在子线程中执行的代码逻辑。
  4. void quit()：终止线程的执行。该方法会发送一个终止信号给线程，通知线程退出。
  5. void wait()：等待线程结束。该方法会阻塞当前线程，直到被调用的线程执行完毕。
  6. bool isRunning() const：检查线程是否正在运行。返回 true 表示线程正在执行中，返回 false 表示线程已经停止。
  7. bool event(QEvent *event)：重写事件处理函数。你可以在子类中重写该函数，处理自定义的事件。
  8. void msleep(unsigned long msecs)：让线程休眠指定的毫秒数。
  9. void setPriority(Priority priority);设置线程优先级

除了上述方法，QThread 类还提供了一些信号，用于线程之间的通信：

  • void started()：线程开始执行时发出的信号。
  • void finished()：线程执行结束时发出的信号。
  • void terminated()：线程被终止时发出的信号，直接退出线程。

需要注意的是，在多线程编程中，Qt 推荐使用信号与槽机制进行线程间的通信，以确保线程安全。可以通过 QObject::connect() 方法将信号连接到槽函数，实现线程间的数据传递和操作。

此外，为了使用 Qt 的事件循环机制处理事件，你可以将 QObject 对象移动到子线程中，然后在子线程中创建和使用这些对象。这样，你就可以使用信号和槽、定时器、事件过滤器等 Qt 提供的功能。注意，使用这种方式时，你需要将 QThread 的对象作为父对象传递给 QObject 对象，以便在父对象销毁时自动销毁子线程。
