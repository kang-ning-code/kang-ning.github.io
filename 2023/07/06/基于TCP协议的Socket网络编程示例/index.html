基于TCP协议的Socket网络编程示例

1、开发环境

开发工具：Qt 5.15

开发语言：C++

2、Qt网络编程

2.1 TCP基本知识

TCP 协议是一种面向连接的、可靠的、基于字节流的传输层通信协议。TCP 通过检验和、序列号、确认应答、重发控制、连接管理以及窗口控制等机制实现可靠性传输。

（1）TCP 通过三次握手来建立可靠的连接。

（2）TCP 四次挥手断开连接。

TCP连接是双向的，在四次挥手中，前两次挥手用于断开一个方向的连接，后两次挥手用于断开另一方向的连接。

（3）TCP报头数据结构

  • 源端口号/目的端口号: 表示数据从哪个进程来, 到哪个进程去.

  • 32位序号:

  • 4位首部长度: 表示该tcp报头有多少个4字节(32个bit)

  • 6位保留: 顾名思义, 先保留着, 以防万一

  • 6位标志位

URG: 标识紧急指针是否有效
ACK: 标识确认序号是否有效
PSH: 用来提示接收端应用程序立刻将数据从tcp缓冲区读走
RST: 要求重新建立连接. 我们把含有RST标识的报文称为复位报文段
SYN: 请求建立连接. 我们把含有SYN标识的报文称为同步报文段
FIN: 通知对端, 本端即将关闭. 我们把含有FIN标识的报文称为结束报文段
原文链接：https://blog.csdn.net/sinat_36629696/article/details/80740678

2.2 QTcpSocket和QTcpServer的基本使用

(1)头文件

#include <QTcpServer>
#include <QTcpSocket>
#include <QHostAddress>

(2)pro文件

要使用 Qt 的网络模块需要在 pro 中加上 network（如果是 VS IDE 就在模块选择里勾选上 network）：

QT += network

(3)基本工作流程

(4)QTcpServer的接口(服务端)

QTcpServer 是 Qt 框架中用于创建 TCP 服务器的类。它提供了一组接口来监听传入的连接请求，并与客户端进行通信。

下面是一些常用的 QTcpServer 接口：

  1. 构造函数和析构函数：
    • QTcpServer(QObject *parent = nullptr)：构造一个 QTcpServer 对象，并可选择将其作为父对象管理。
    • ~QTcpServer()：析构函数，释放 QTcpServer 对象及其资源。
  2. 服务器操作：
    • bool listen(const QHostAddress &address = QHostAddress::Any, quint16 port = 0)：开始监听指定的地址和端口上的连接请求。
    • void close()：停止监听连接请求并关闭服务器。
    • bool isListening() const：检查服务器是否正在监听连接请求。
  3. 客户端连接管理：
    • bool hasPendingConnections() const：检查是否有待处理的连接请求。如果有调用appendToSocketList(QTcpSocket* socket)
    • QTcpSocket *nextPendingConnection()：返回下一个待处理连接请求的 QTcpSocket 对象。
    • void pauseAccepting()：暂停接受新的连接请求。
    • void resumeAccepting()：恢复接受新的连接请求。
  4. 信号和槽：
    • void newConnection()：当有新的连接请求时发出的信号。

在接受新的连接请求后，您可以使用 nextPendingConnection() 返回的 QTcpSocket 对象来与客户端进行通信。QTcpSocket 提供了发送和接收数据的接口，可以使用它们进行数据交换。

注意，返回的 QTcpSocket 对象不能在另一个线程使用，如果需要在别的线程管理这个 socket 连接，需要重写 Server 的 incomingConnection() ，将 sokcet 描述符传递给别的线程并创建 QTcpSocket：

void QTcpServer::incomingConnection(qintptr socketDescriptor)

您可以参考 Qt 的官方文档以获取更详细的信息和示例代码：https://doc.qt.io/qt-6/qtcpserver.html

(5)QTcpSocket的接口(客户端)

QTcpSocket 是 Qt 框架中的一个类，用于在 TCP/IP 网络上进行通信。它提供了一组接口来建立、发送和接收 TCP 连接。

下面是一些常用的 QTcpSocket 接口：

  1. 构造函数和析构函数：
    • QTcpSocket(QObject *parent = nullptr)：构造一个 QTcpSocket 对象，并可选择将其作为父对象管理。
    • ~QTcpSocket()：析构函数，释放 QTcpSocket 对象及其资源。
  2. 连接管理：
    • void QAbstractSocket::connectToHost(const QString &hostName, quint16 port)：连接到指定的主机和端口。//客户端
    • void QAbstractSocket::disconnectFromHost()：断开与主机的连接。
    • bool QAbstractSocket::isConnected() const：检查是否已经连接到主机。
  3. 数据发送和接收：
    • qint64 QIODevice::write(const char *data, qint64 size)：将指定大小的数据发送到已连接的主机。
    • qint64 QIODevice::read(char *data, qint64 maxSize)：从连接中读取最多 maxSize 字节的数据。
    • QByteArray QIODevice::readAll()：读取连接中的所有可用数据。
  4. 错误处理：
    • QAbstractSocket::SocketError error() const：获取最后一次发生的错误。
    • QString errorString() const：获取描述错误的字符串。
  5. 信号和槽：
    • void QIODevice::readyRead()：当有新的数据可用时发出的信号。
    • void connected()：当成功连接到主机时发出的信号。
    • void disconnected()：当与主机的连接断开时发出的信号。
    • void error(QAbstractSocket::SocketError socketError)：当出现错误时发出的信号。

这些只是 QTcpSocket 类的一些常用接口，还有其他一些接口和功能可用于更高级的网络通信操作。您可以参考 Qt 的官方文档以获取更详细的信息和示例代码：https://doc.qt.io/qt-6/qtcpsocket.html
